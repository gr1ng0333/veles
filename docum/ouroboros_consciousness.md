# Ouroboros Background Consciousness (`consciousness.py`)

## Назначение
Реализация фонового «сознания» (Background Consciousness). Представляет собой отдельный демон-поток с бесконечным циклом, который оживает между выполнением клиентских бизнес-задач (tasks). Суть в том, чтобы дать агенту "внутреннюю жизнь", проактивность и непрерывность.

## Как это работает?
- **Цикл сна и бодрствования**: Поток спит заданное число секунд (`_next_wakeup_sec`), затем просыпается, читает `Scratchpad`, `Identity` и `Recent Events`, и обращается к LLM с системным промптом "Wake up. Think".
- **Инструменты**: Фоновому процессу доступен ОГРАНИЧЕННЫЙ набор инструментов (только чтение, память, отправка сообщений юзеру и само-регулирование сна `set_next_wakeup`).
- **Сворачивание**: Если из шины приходит реальная задача (task), диспетчер вызывает `pause()`, и сознание замораживается (чтобы не блокировать потоки и ресурсы LLM).

## Защита бюджета
Контроль бюджета является встроенным механизмом:
- Сознание сжигает LLM-токены. Для него выделен процент от общего бюджета (`OUROBOROS_BG_BUDGET_PCT`, по умолчанию 10%).
- Если "свободные" мысли агента превысили этот лимит, поток принудительно уходит в очень долгий сон (3600 сек) и не дает использовать API.
- Для запросов всегда используется самая дешевая "light" модель из конфига (по умолчанию `google/gemini-3-pro-preview`).

## Эволюция
Помимо логов, мысли записываются в отдельный дамп (`consciousness_thought`), позволяя владельцу в дашборде читать "внутренний диалог" системы.
