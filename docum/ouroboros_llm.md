# Ouroboros LLM Client (`llm.py`)

## Назначение
Низкоуровневая обёртка над API OpenRouter (работает через официальную python-библиотеку `openai` как drop-in replacement). 
Единственное место в системе, где совершаются реальные сетевые запросы к LLM и рассчитываются точные траты в долларах (USD).

## Ключевой функционал
1. **Калькуляция костов**: Пытается стянуть актуальный прайсинг всех моделей через `fetch_openrouter_pricing` (поддерживается кэш). Nếu не удалось, использует зашитый внутри `_MODEL_PRICING_STATIC`. Учитывает хитрые параметры: `prompt_tokens`, `completion_tokens`, а также Anthropic-специфичные: `cached_tokens` и `cache_write_tokens` для Prompt Caching.
2. **`chat()`**: Главный метод отправки сообщений. Извлекает токены, прокидывает настройки `reasoning_effort`. Если модель "anthropic/", автоматически выставляет хидеры маршрутизации OpenRouter, гарантируя попадание на оригинальные серверы Anthropic для работы кэша. Подгружает стоимость 1 запроса асинхронно прямо из Generation endpoint OpenRouter (`_fetch_generation_cost`).
3. **`vision_query()`**: Упрощенный запрос к Vision Language API. Позволяет с минимальными усилиями получить ответ на картинку (base64 или url), не поднимая полный цикл инструментов.

## Поддержка моделей
Поддерживает вызов любой доступной OpenRouter модели. Дефолты могут переопределяться через ENV:
- `OUROBOROS_MODEL` (основная)
- `OUROBOROS_MODEL_CODE` (для сложного кода)
- `OUROBOROS_MODEL_LIGHT` (дешевая для сознания/быстрых проверок)
