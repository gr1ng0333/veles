# Ouroboros LLM Loop (`loop.py`)

## Назначение
Сердце движка Ouroboros. Файл содержит бизнес-логику общения с LLM API (`run_llm_loop`). Собирает контекст, стучится в LLM, парсит ответы, вызывает нужные инструменты, и всё это гоняет по кругу, пока LLM не скажет финальный ответ (не перестанет выдавать `tool_calls`).

## Роутинг инструментов и таймауты
Если LLM вызывает несколько инструментов параллельно (`tool_calls > 1`), движок проверяет, являются ли они безопасными инструментами только для чтения (whitelist: `repo_read`, `repo_list`, `codebase_digest` и т.д). 
- Если **да** — они исполняются по-настоящему параллельно через `ThreadPoolExecutor`. 
- Если среди них есть инструменты модификации, они выполняются последовательно для защиты стейта.

Все вызовы инструментов оборачиваются в **жесткие таймауты** (взятые из схемы инструмента). Если зависает, поток убивается, а LLM получает системное сообщение `⚠️ TOOL_TIMEOUT`.

## Stateful Playwright
Особое внимание уделяется браузеру (`_StatefulToolExecutor`). Модули Playwright не любят разные потоки. `loop.py` использует "липкий поток" (thread-affinity) для всех браузерных инструментов (`browse_page`, `browser_action`), чтобы избежать крэша Greenlet. При таймауте этот "липкий поток" сбрасывается и пересоздается.

## Инъекция событий и самопроверка (Self-Check)
1. **Динамический тулинг**: Умеет во время общения инжектировать в список используемых `tool_schemas` новые, которые LLM сам попросил включить (через мета-тулзы).
2. **Вживление сообщений (Drain)**: Между вызовами тулов читает сообщения, присланные юзером "вдогонку", и подкидывает их в контекст, чтобы LLM мог изменить свое поведение.
3. **Checkpoint (Когнитивная пауза)**: Каждые 50 ходов в рамках задачи принудительно отправляет агенту системное предупреждение `[CHECKPOINT]`, предлагая притормозить, посмотреть в бюджет и прекратить крутиться в бесконечных циклах.
4. **Бюджетный тормоз**: Принудительно отбивает задачу (Hard stop), если она сожрала больше 50% от оставшегося глобального бюджета.

## Сжатие контекста
Прямо в рантайме отслеживает объем истории. 
- На 8 ходу обрезает старые ответы тулзов, оставляя только их превью (через функцию `compact_tool_history` из `context.py`). 
- Обрабатывает ручные запросы агента на сжатие (`compact_tool_history_llm`).
