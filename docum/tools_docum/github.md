# GitHub Tools (`github.py`)

## Назначение
Инструменты для взаимодействия с GitHub Issues. Позволяют агенту читать, создавать, комментировать и закрывать задачи/баги (issues) прямо в репозитории проекта. 

Используют под капотом CLI утилиту `gh` (требует установленного `gh` и авторизации). Репозиторий вычисляется автоматически локально (`gh repo view`) или берется из переменных окружения `GITHUB_USER`/`GITHUB_REPO`.

## Безопасность (Security)
Все вызовы `gh` CLI спроектированы так, чтобы текст пользователя/агента (например, body документа или title) передавался через STDIN (`--body-file -`), либо строго через флаги (`--flag=value`). Это предотвращает уязвимости типа argument injection. Ошибки вывода STDERR намеренно обрезаются до 200 символов, чтобы не допустить утечек.

## Доступные инструменты (Tools)

### `list_github_issues`
- **Описание**: Получить список всех Issues выбранного статуса и меток. Выводит в виде Markdown (номер, заголовок, автор, метки и превью 200 символов описания).
- **Параметры**:
  - `state` (str): `open`, `closed`, `all` (по умолчанию: `open`).
  - `labels` (str): Список меток через запятую.
  - `limit` (int): Максимум элементов (по умолчанию: 20, жесткий лимит внутри функции: 50).

### `get_github_issue`
- **Описание**: Получить полную информацию о конкретном Issue (включая всё тело, статус и до 10 последних комментариев).
- **Параметры**:
  - `number` (int, required): Номер Issue (напр. `42`).

### `comment_on_issue`
- **Описание**: Оставить комментарий в Issue от имени бота.
- **Параметры**:
  - `number` (int, required): Номер Issue.
  - `body` (str, required): Текст комментария в формате Markdown.

### `close_github_issue`
- **Описание**: Закрыть Issue. При желании можно приложить финальный (closing) комментарий.
- **Параметры**:
  - `number` (int, required): Номер Issue.
  - `comment` (str): Опциональный комментарий перед закрытием.

### `create_github_issue`
- **Описание**: Создать новый Issue для отслеживания задачи, документирования бага или планирования функциональности. Если указаны метки, они применяются к Issue отдельным вызовом.
- **Параметры**:
  - `title` (str, required): Заголовок.
  - `body` (str): Основной текст (Markdown).
  - `labels` (str): Метки через запятую.
