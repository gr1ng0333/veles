# Knowledge Base (`knowledge.py`)

## Назначение
Инструменты для долгосрочного хранения информации агентом. Механизм Knowledge Base сохраняет структурированную память агента прямо на Google Drive в папке `memory/knowledge/`. Подходит для хранения паттернов кода, рецептов (recipes), списков ошибок или нюансов проектов. 

Модуль автоматически создает и обновляет индекс `_index.md`, содержащий список всех топиков (файлов) с выжимкой их содержимого (до 3 первых смысловых строк).

## Защита
Названия топиков санитарятся через регулярные выражения для блокировки инъекций путей (`../`, зарезервированные имена).

## Доступные инструменты (Tools)

### `knowledge_read`
- **Описание**: Чтение сохраненного markdown-файла с базой знаний.
- **Параметры**:
  - `topic` (str, required): Название топика (только буквы, цифры, дефис, точка, подчеркивание).

### `knowledge_write`
- **Описание**: Запись или дополнение (append) топика. При записи `knowledge.py` интеллектуально обновляет глобальный `_index.md` для этого топика.
- **Параметры**:
  - `topic` (str, required): Имя топика.
  - `content` (str, required): Содержимое в формате Markdown.
  - `mode` (str): Режим записи: `overwrite` (по умолч.) или `append`.

### `knowledge_list`
- **Описание**: Возвращает список всех доступных топиков из индексного файла. Если индекс отсутствует — автоматически собирает его.
- **Параметры**: Нет.
