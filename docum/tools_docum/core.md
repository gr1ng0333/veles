# Core Tools (`core.py`)

## Назначение
Набор базовых инструментов для работы с файловой системой (репозиторий и Google Drive), а также для анализа кодовой базы и создания дайджестов диалогов.

## Доступные инструменты (Tools)

### `repo_read`
- **Описание**: Чтение текстового файла (UTF-8) из директории репозитория.
- **Параметры**: 
  - `path` (str, required): Относительный путь к файлу.

### `repo_list`
- **Описание**: Получение списка файлов и папок в репозитории (до 500 элементов).
- **Параметры**:
  - `dir` (str): Относительный путь к папке (по умолчанию `.`).
  - `max_entries` (int): Максимальное количество элементов для возврата.

### `drive_read`
- **Описание**: Чтение файла из структуры Google Drive.
- **Параметры**: 
  - `path` (str, required): Путь относительно корневой папки MyDrive/Ouroboros.

### `drive_list`
- **Описание**: Список файлов в директории Drive.
- **Параметры**: Аналогичны `repo_list`.

### `drive_write`
- **Описание**: Запись строки в файл на Drive (перезапись или добавление).
- **Параметры**:
  - `path` (str, required): Путь к файлу.
  - `content` (str, required): Содержимое.
  - `mode` (str): `overwrite` (по умолчанию) или `append`.

### `send_photo`
- **Описание**: Отправка изображения (Base64) в Telegram чат владельца. Интегрируется с состоянием браузера — можно передать `__last_screenshot__`, чтобы отправить последний скриншот из `browse_page` или `browser_action`.
- **Параметры**:
  - `image_base64` (str, required): Base64 PNG.
  - `caption` (str): Подпись к фотографии.

### `codebase_digest`
- **Описание**: Возвращает компактный дайджест всей кодовой базы. Проходится по директории, пропуская скрытые и служебные папки (например, `.git`, `__pycache__`, `venv`). Для Python файлов использует `ast` для извлечения имен классов и функций. Возвращает форматированную сводку с размерами файлов и перечнем их структурных элементов.
- **Параметры**: Нет.

### `summarize_dialogue`
- **Описание**: Суммаризация истории диалога (логов) с помощью LLM (модель Light). Формирует выжимку (ключевые решения, предпочтения, паттерны) за последние `N` сообщений и сохраняет результат в `memory/dialogue_summary.md` на Drive.
- **Параметры**: 
  - `last_n` (int): Количество последних сообщений (по умолчанию 200).
- **Дополнительно**: Интегрирован с механизмом расчета бюджетов (отправляет событие `llm_usage` в очередь).

### `forward_to_worker`
- **Описание**: Инструмент для пересылки сообщений запущенной фоновой задаче (worker). Добавляет в ее `task_id` специальное уведомление, чтобы воркер увидел его в своем ближайшем LLM-цикле (`owner_inject`).
- **Параметры**:
  - `task_id` (str, required): ID задачи воркера.
  - `message` (str, required): Текст сообщения.
