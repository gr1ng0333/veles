# Ouroboros Tools Documentation

Эта папка содержит подробную техническую документацию по всем доступным агенту Ouroboros инструментам (Tools). Модули Ouroboros построены по плагинной архитектуре — чтобы добавить новый инструмент, достаточно создать Python-файл в `ouroboros/tools/` и экспортировать функцию `get_tools() -> List[ToolEntry]`.

## Индекс модулей инструментов:

1. **[Ядро и Файловая система (Core Tools)](./core.md)**
   Инструменты для работы с диском, репозиторием (чтение/список), выгрузки дайджестов кодовой базы и суммаризации истории: `repo_read`, `repo_list`, `codebase_digest`, `send_photo`, и другие.

2. **[Реестр инструментов (Registry)](./registry.md)**
   Архитектура реестра инструментов: объекты Context (`ToolContext`) и Entry (`ToolEntry`), управляющие доступом и жизненным циклом (SSOT).

3. **[Контроль состояния (Control Tools)](./control.md)**
   Мета-инструменты агента: управление задачами (расписание, отмена, результаты), переключение версий LLM моделей и уровней усилий (`switch_model`), ведение дневника/scratchpad, перезапуск среды и прямые отправки сообщений владельцу.

4. **[Обозреватель сети (Browser Tools)](./browser.md)**
   Плотная работа с Playwright: навигация по браузеру, сбор текста/html/markdown, клики, ввод текста, выполнение JS в консоли веб-страницы и сохранение скриншотов (`browse_page`, `browser_action`).

5. **[Git и Версионирование (Git Tools)](./git.md)**
   Декларативные коммиты и безопасные пуши (`repo_write_commit`, `repo_commit_push`). Включает в себя механизм блокировок `git.lock` во избежание конфликтов между параллельными задачами агента.

6. **[GitHub интеграция (GitHub Tools)](./github.md)**
   Набор функций над `gh` CLI для прямого взаимодействия с задачами GitHub из агента: получение списков открытых багов, комментирование, закрытие и создание задач.

7. **[Компактизация контекста (Compact Context)](./compact_context.md)**
   LLM-ориентированный механизм выборочной очистки/сокращения контекста (истории вызовов инструментов). Агент сам вызывает его, если чувствует разрастание промпта, прося систему сжать старые ответы.

8. **[Управление шеллом и Claude CLI (Shell Tools)](./shell.md)**
   Один из важнейших модулей: `run_shell` для прямого вызова bash/powershell команд и `claude_code_edit` для делегирования массивных правок кода или рефакторинга в нативную утилиту `Claude Code CLI`.

9. **[Подхват доп. инструментов (Tool Discovery)](./tool_discovery.md)**
   Мета-инструмент для самостоятельного обнаружения (`list_available_tools`) и включения (`enable_tools`) расширенных (non-core) функций `Ouroboros`, которых нет в основном промпте.

10. **[Анализ зрения (Vision Tools)](./vision.md)**
    Интеграция с Vision Language Models для оценки нарисованных графиков или скриншотов браузера (показывает где ошибки вёрстки).

11. **[Рецензирование кода (Multi-model Review)](./review.md)**
    Синхронный запуск консенсус-запросов к N количеству LLM-моделей (через OpenRouter) для аудита или проверки архитектурных решений агента на прочность. 

12. **[База знаний (Knowledge Base)](./knowledge.md)**
    Долговременная память на Google Drive. Позволяет агенту записывать, читать и дополнять документы (рецепты, правила конкретного проекта) в markdown файлах (топиках), с автогенерацией `_index.md`.

13. **[Статистика эволюции (Evolution Stats)](./evolution_stats.md)**
    Генератор таймлапса эволюции: автоматически обходит git историю и считает число строк кода / длину BIBLE (размер "философии") / длину SYSTEM ("сущности") и загружает в веб-дайджест `evolution.json`.

14. **[Здоровье проекта (Codebase Health)](./health.md)**
    Инструмент, считающий внутреннюю статику проекта (АСТ). Предупреждает о "вирусоподобном" разрастании файлов (>1000 строк) или функций (>150 строк), нарушающем принцип 5 философии Ouroboros.
