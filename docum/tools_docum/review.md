# Multi-Model Review (`review.py`)

## Назначение
Продвинутый инструмент Ouroboros для самопроверок и ревью сложного кода через механизм консенсуса нескольких моделей. Отправляет запрос (prompt) параллельно сразу в несколько заданных LLM-моделей (до 10) через API `OpenRouter`. 

Каждый ответ парсится на наличие слов "PASS" или "FAIL" (на первых 3х строчках). Функция автоматически логирует потребленные ресурсы (`llm_usage` event) для учета бюджета агента.

## Особенности
Асинхронная архитектура под капотом. Так как ToolRegistry работает синхронно, инструмент оборачивает asyncio запросы (с лимитом параллелизма `CONCURRENCY_LIMIT = 5`) в потоки (ThreadPoolExecutor), чтобы не блокировать loop Ouroboros.

Поддерживаются тайм-ауты (120с на запрос) и защита от падений API.

## Доступные инструменты (Tools)

### `multi_model_review`
- **Описание**: Отправить заданный кусок текста/кода на агрегированное ревью нескольким моделям.
- **Параметры**:
  - `content` (str, required): Код или текст для ревью.
  - `prompt` (str, required): Точные инструкции для ревьюеров (что искать, какие критерии).
  - `models` (array of strings, required): Массив OpenRouter ID моделей (например, `["openai/o3-mini", "google/gemini-2.5-pro"]`).
