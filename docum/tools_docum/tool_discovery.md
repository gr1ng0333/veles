# Tool Discovery (`tool_discovery.py`)

## Назначение
Мета-инструменты Ouroboros. Позволяют агенту (LLM) самостоятельно узнавать о наличии расширенных (non-core) инструментов и включать их только при необходимости. 
Механизм реализован через паттерн динамической активации, что позволяет снизить когнитивную нагрузку на LLM – в базовом промпте лежат только самые необходимые инструменты (core tools), а остальные подтягиваются через "Discovery".
Оригинальные дефолтные загрузки обработчиков в `core.py` или `loop.py` инжектируют модуль глобального реестра: `_registry`.

## Доступные инструменты (Tools)

### `list_available_tools`
- **Описание**: Выводит список всех дополнительных (non-core) инструментов, которых **нет** в текущем наборе активных инструментов. Возвращает форматированный список Markdown с их названиями и краткими описаниями.
- **Параметры**: Нет. Не требует аргументов.
- **Использование**: Агент может вызвать его, если чувствует, что стандартных тулзов не хватает для текущей задачи.

### `enable_tools`
- **Описание**: Включает один или несколько дополнительных инструментов по их названию. Их схемы будут добавлены в текущий контекст (`ToolContext`) и будут доступны агенту до завершения текущей задачи.
- **Параметры**:
  - `tools` (str, required): Строка с названиями инструментов, разделенными запятыми (например, `multi_model_review,generate_evolution_stats`). Возвращает статус (какие из запрошенных были найдены и активированы, а какие нет). 
- **Использование**: После вызова функции, LLM в следующем цикле получит обновленный `function list`.
