# Evolution Stats (`evolution_stats.py`)

## Назначение
Независимый "дополнительный" (non-core) инструмент, который генерирует данные для таймлапса эволюции (Evolution Time-Lapse) и выводит их на дашборд (вкладка Evolution в `app.html`). Инструмент собирает метрики роста кодовой базы на основе git истории репозитория.

Оценивается по трём осям (осям развития Ouroboros):
1. **Technical**: `py_lines` — Количество строк Python кода;
2. **Philosophical**: `bible_bytes` — Размер файла `BIBLE.md` (набор правил);
3. **Self-Concept**: `system_bytes` — Размер промпта `SYSTEM.md` (внутреннее представление о себе).

## Подробности реализации
- Сбор статистики идёт через вызовы `git log`, выборку N (до 100) равномерно распределенных коммитов. 
- Для каждого сэмпла вызывается `git ls-tree` и подсчитывается точное количество строк и размер нужных файлов на момент создания коммита.
- Автоматически инжектирует вкладку "Evolution" в файл `app.html` веб-приложения (с помощью JavaScript и Chart.js).
- Сгенерированный JSON-статус заливается (`push`) прямо в папку `docs/evolution.json` используя GitHub API ключи из окружения (переменная окружения `GITHUB_TOKEN`).

## Доступные инструменты (Tools)

### `generate_evolution_stats`
- **Описание**: Функция-обертка для запуска процесса сбора и заливки эволюционных данных на дашборд. Ввиду большой истории коммитов может занимать 15-30 секунд. Возвращает строку с краткой сводкой результатов (размер файлов на актуальный коммит).
- **Параметры**: Отсутствуют.
