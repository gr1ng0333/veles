# Compact Context (`compact_context.py`)

## Назначение
Инструмент для ручного (со стороны агента) контроля над переполнением контекста диалога. Служит для сжатия старых запросов-ответов инструментов в текущей сессии для экономии токенов промпта. Реализует принцип «LLM-first»: агент сам решает, когда и в какой степени нужно сжать историю, а не полагается на жесткий лимит.

## Доступные инструменты (Tools)

### `compact_context`
- **Описание**: Сигнализирует петле (loop) о необходимости компактизации цепочки истории вызовов инструментов. Не удаляет данные бесследно — старые раунды сворачиваются в однострочные "Summary".
- **Особенности**: 
  - Вместо немедленного выполнения, сохраняет флаг `_pending_compaction` в объекте контекста `ctx`. Фактическая компактизация происходит внутри функции главного цикла (loop).
- **Параметры**:
  - `keep_last_n` (int): Количество последних полных (нетронутых) вызовов инструментов, которые нужно оставить "как есть". По умолчанию: `6`. Допустимый диапазон: `2 - 20`. Меньше значение = сильнее сжатие.
