# Ouroboros Tool Registry (`registry.py`)

## Назначение
Единая точка входа (SSOT) для всех инструментов (tools) в системе Ouroboros. Отвечает за сбор всех доступных инструментов, предоставление их схем (для LLM) и их выполнение (`execute`). 

## Архитектура
Плагинная архитектура: каждый модуль в папке `tools/` должен экспортировать функцию `get_tools() -> List[ToolEntry]`. `ToolRegistry` автоматически загружает все модули при инициализации.

## Основные классы

### `ToolContext`
Контекст выполнения инструмента, который передается агентом перед каждой задачей. 
- **Атрибуты**: 
  - `repo_dir`, `drive_root`: Базовые пути к репозиторию и Google Drive.
  - `branch_dev`: Текущая ветка разработки.
  - `pending_events`: Список событий, собранных в течение задачи (очередь событий).
  - `current_chat_id`, `current_task_type`, `last_push_succeeded`: Статусы текущего состояния агента.
  - `browser_state`: Инкапсулированное состояние браузера (экземпляр `BrowserState`).
  - `active_model_override`, `active_effort_override`: Перехваты параметров модели LLM.
- **Методы**: 
  - `repo_path()`, `drive_path()`, `drive_logs()`: Возвращают разрешенные (resolved) пути относительно базовых директорий.

### `ToolEntry`
Описание отдельного инструмента (тулзы).
- **Атрибуты**:
  - `name`: Имя инструмента.
  - `schema`: JSON схема для LLM.
  - `handler`: Функция-обработчик с сигнатурой `fn(ctx: ToolContext, **args) -> str`.
  - `is_code_tool`: Флаг, указывает, относится ли инструмент к специфичным операциям с кодом.
  - `timeout_sec`: Допустимое время выполнения (по умолчанию 120 сек).

### `ToolRegistry`
Главный реестр инструментов.
- **Особенности**:
  - `CORE_TOOL_NAMES`: Жестко заданный список базовых инструментов, всегда доступных для LLM.
  - Авто-опрос модулей (`_load_modules()`) в `ouroboros/tools/*`.
  - Интеграция с подменой обработчиков (`override_handler`) для инъекции замыканий (например, для контроля доступа к мета-состоянию).

## Ключевые методы `ToolRegistry`
- `schemas(core_only: bool)`: Возвращает список схем для LLM (либо только базовых, либо всех зарегистрированных).
- `list_non_core_tools()`: Список дополнительных инструментов (название и описание).
- `execute(name: str, args: Dict) -> str`: Безопасное выполнение инструмента с перехватом исключений (`TypeError`, `Exception`).
