# Git Tools (`git.py`)

## Назначение
Базовые инструменты (core) для взаимодействия с Git. Осуществляют атомарные коммиты и безопасные пуши файлов в девелоперскую ветку (`branch_dev`). Реализуют механизмы защиты от конкурентного доступа к репозиторию (`git.lock`) и (опционально) запускают тесты перед пушем изменений (`pre-push test gate`).

Инструменты в этом модуле строго помечены как `is_code_tool=True`.

## Архитектура блокировок
- `_acquire_git_lock()`, `_release_git_lock()`: Так как Ouroboros может вызывать шелл/коммиты из разных фоновых задач одновременно, используется механизм блокировки через создание файлового лока `git.lock` во избежание сбоев в `git index`.

## Пред-релизные тесты
Функция `_git_push_with_tests()` проверяет наличие переменной окружения `OUROBOROS_PRE_PUSH_TESTS`. Если она активна и в репозитории есть папка `tests/`, перед выполнением `git push` модуль сначала запустит `pytest tests/...`. При падении тестов коммит отменяется и сохраняется только локально.

## Доступные инструменты (Tools)

### `repo_write_commit`
- **Описание**: Записать один файл целиком и сразу сделать коммит с пушем в ветку `ouroboros`. Подходит для простых детерминированных исправлений.
- **Параметры**:
  - `path` (str, required): Относительный путь к файлу в репозитории.
  - `content` (str, required): Полное содержимое файла для перезаписи.
  - `commit_message` (str, required): Сообщение коммита.

### `repo_commit_push`
- **Описание**: Выполнить коммит и пуш уже измененных файлов (например, после `claude_code_edit` или `run_shell`). Перед пушем делает `pull --rebase`.
- **Параметры**:
  - `commit_message` (str, required): Сообщение к коммиту.
  - `paths` (array of strings, optional): Конкретные пути к файлам для `git add`. Если не переданы, добавляются все отслеживаемые и неотслеживаемые изменения (вызов `git add -A`). При передаче `paths` модуль также предупреждает агента, если остались какие-то `untracked files`.

### `git_status`
- **Описание**: Мгновенный показ статуса `git status --porcelain`.
- **Параметры**: Нет.

### `git_diff`
- **Описание**: Показ различий с помощью `git diff`.
- **Параметры**:
  - `staged` (boolean): По умолчанию `False`. Передать `true`, если требуется просмотреть внесенные изменения (`git diff --staged`) перед коммитом.
