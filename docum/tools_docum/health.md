# Codebase Health (`health.py`)

## Назначение
Небольшой инструмент для самодиагностики кодовой базы Ouroboros. Помогает агенту следить за "здоровьем" проекта и соответствием заложенным в BIBLE.md принципам (особенно Principle 5: Minimalism).

Под капотом импортирует функции анализа AST из модуля `review.py` (`collect_sections`, `compute_complexity_metrics`).

## Метрики, которые собираются
- Общее количество файлов, строк, функций и символов.
- Самые большие файлы в проекте (предупреждение `⚠️ OVERSIZED` выдается для файлов > 1000 строк кода).
- Самые длинные функции (предупреждение `⚠️ OVERSIZED` выдается для функций > 150 строк).
- Сводка нарушений принципов "Bible" (выводит конкретные файлы/функции и их размеры, если они нарушают пределы).

## Доступные инструменты (Tools)

### `codebase_health`
- **Описание**: Собирает метрики и возвращает структурированный Markdown-отчет. Рекомендуется использовать для периодического селф-ревью.
- **Параметры**: Нет. Не требует аргументов.
