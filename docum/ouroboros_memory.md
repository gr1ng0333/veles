# Ouroboros Memory (`memory.py`)

## Назначение
Слой абстракции над Google Drive для управления долговременной памятью агента Ouroboros (папка `memory/` и `logs/`). Не зависит от LLM, содержит только файловые операции чтения, записи и парсинга JSONL.

## Ключевые файлы
1. **Scratchpad** (`scratchpad.md`): Оперативный блокнот агента для планов и записок.
2. **Identity** (`identity.md`): Персистентная сущность агента (кто я, какие выводы сделал из прошлых неудач).
3. **Journal** (`scratchpad_journal.jsonl`): Журнал изменений.

## Стуруктурированная нарезка JSONL
Так как все логи Ouroboros (и его supervisor диспетчера) пишутся в единые JSONL-стримы (chat, events, tools, progress), `memory.py` содержит набор методов `summarize_XXX`.
Всё это используется `context.py` для того, чтобы сжать последние 200 строк JSONL-файла в короткую текстовую "выжимку" (хронологию), которую LLM легко прочитает и поймёт, чем он сейчас занимался в текущей сессии.

## Особенности
Никогда не обрезает сообщения Владельца. Ограничивает превью системных мыслей и выводов тулзов до 800 символов.
